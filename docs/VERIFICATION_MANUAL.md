# 系統驗證手冊 (Verification Manual)

> **用途：** 提供可執行的測試步驟，驗證系統各項功能是否正常運作。
> **適用版本：** v1.6（Phase 1+2+3+4 全部完成）
> **對應文件：** SDD v1.6、DEVELOPMENT_ROADMAP.md、CHANGELOG.md

---

## 使用方式

1. 每個測試案例依序執行「操作步驟」
2. 對照「預期結果」確認功能正確
3. 若實際結果不符，記錄於「失敗說明」並回報

測試案例編號規則：`V<Phase>.<功能>.<序號>`（例：`V1.1.01` = Phase 1 第 1 項功能的第 1 個測試）

## 前置作業

### P.01 首次部署後建立 SuperAdmin 帳號
| 步驟 | 操作 |
|------|------|
| 1 | 開啟部署後的網站首頁 |
| 2 | 打開瀏覽器 DevTools → Console |
| 3 | 執行：`fetch('/.netlify/functions/api', { method:'POST', headers:{'Content-Type':'application/json'}, body:JSON.stringify({action:'initialize-database'}) }).then(r=>r.json()).then(console.log)` |
| 4 | 觀察 Console 輸出 |

**預期結果：**
- 首次執行：`{ body: "{\"message\":\"已建立 SuperAdmin 帳號...\"}" }`
- 再次執行：`{ body: "{\"message\":\"已初始化\"}" }`

**失敗說明：**
- 若回傳 500 → 檢查 Netlify Functions 日誌
- 若 ADMIN 帳號仍無法登入 → 檢查 Firestore `employees/ADMIN` 是否存在且 `password` 欄位包含 `:`

## 前後比對測試（v1.0 → v1.3）

| 項目 | v1.0 行為（有問題） | v1.3 行為（已修正） |
|------|--------------------|---------------------|
| 初始化 | 自動寫入 5 位假員工+假人名排班 | 只建立 ADMIN + 空白模板 |
| 排班修改 | 改單日會連動所有同星期 | 只影響該日 |
| 班別時段 | 前端不可編輯 | Modal 可編輯起訖時間 |
| 管理員打卡 | 無此功能 | 側邊欄「我的打卡」 |
| IP 驗證 | 寫死 127.0.0.1 | 後端取真實 IP |
| 密碼儲存 | Firestore 明文 | scrypt 雜湊 |
| 密碼強度 | 4 字元即可 | 8 字元 + 英數 |
| 登入失敗 | 無限制 | 5 次鎖 15 分鐘 |
| 重設密碼 | alert 顯示明文 | 不顯示，只通知員工 |
| 薪資權限 | 所有 Admin 可看 | 僅 SuperAdmin |
| 操作紀錄 | 無 | auditLogs 全記錄 |
| 休館值班 | 無法安排 | 休館(值班) 狀態 |
